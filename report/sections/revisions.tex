\section{Ревизии}

В данном разделе для каждой ревизии отдельно показаны команды, которые выполняются пользователями индивидуально, чтобы воспроизвести структуру блок-схемы, выданной по варианту.

\subsection{Ревизия №0}

Листинг Bash-скрипта для системы \texttt{Git}:
\begin{lstlisting}
#receive updates from repository for remote branches
git -C ./output/git_project/local/Red fetch origin
#rename current branch
git -C ./output/git_project/local/Red branch -m br-0
#apply desired changes
cp -rf ./output/git_project/diff_cache/0/. ./output/git_project/local/Red
#schedule commiting all applied changes
git -C ./output/git_project/local/Red add -A
#save scheduled changes in the revision
git -C ./output/git_project/local/Red commit -m "r0"
#send changes to the remote repository
git -C ./output/git_project/local/Red push -u origin br-0
\end{lstlisting}

Листинг Bash-скрипта для системы \texttt{Subversion}:
\begin{lstlisting}
#bring changes from repository
svn --username Red update ./output/svn_project/local/Red
svn --username Red copy file:///project/output/svn_project/remote/trunk file:///project/output/svn_project/remote/branches/br-0 -m "create a new branch br-0"
#change local branch
svn --username Red switch file:///project/output/svn_project/remote/branches/br-0 ./output/svn_project/local/Red
#apply desired changes
cp -rf ./output/svn_project/diff_cache/0/. ./output/svn_project/local/Red
#schedule addition of files if there are new ones
svn --username Red add --force ./output/svn_project/local/Red
#save done changes on branch
svn --username Red commit -m "r0" ./output/svn_project/local/Red
\end{lstlisting}

#receive updates from repository for remote branches
git -C ./output/git_project/local/Blue fetch origin
#sync state of the desired branch with the remote repository
git -C ./output/git_project/local/Blue branch -f br-2 origin/br-2
#apply changes from branch into single commit, preserving remote branch
git -C ./output/git_project/local/Blue merge br-2 --no-ff -m "merge commit"
# Auto-merging Lab4.java
# CONFLICT (content): Merge conflict in Lab4.java
# Automatic merge failed; fix conflicts and then commit the result.
git -C ./output/git_project/local/Blue diff --name-only --diff-filter=U
# Lab4.java
#having problems with merging, trying to solve them...
cp -rf ./output/git_project/diff_cache/8/. ./output/git_project/local/Blue
#schedule commiting all applied changes
git -C ./output/git_project/local/Blue add -A
#save scheduled changes in the revision
git -C ./output/git_project/local/Blue commit -m "merge commit"
#send changes to the remote repository
git -C ./output/git_project/local/Blue push -u origin br-1
\end{lstlisting}

#bring changes from repository

svn --username Blue status ./output/svn_project/local/Blue

svn --username Blue resolve --accept working -R ./output/svn_project/local/Blue
#save done changes on branch
svn --username Blue commit -m "merge commit" ./output/svn_project/local/Blue

\subsubsection{Разрешение конфликтов}

В процессе создания коммита в \texttt{Git} возникли следующие конфликты:
\begin{lstlisting}
#viewing conflicts of ./output/git_project/local/Blue/Lab4.java
Line 22: <<<<<<< HEAD
Line 23: previous : 3
Line 24: =======
Line 25: previous : 5
Line 26: >>>>>>> br-2
\end{lstlisting}

Чтобы их разрешить, необходимо выбрать одну из предложенных версий изменений. Сверимся с предоставленным снимком репозитория и внесём следующие изменения:

\begin{lstlisting}
--- ./output/git_project/local/Blue/Lab4.java	2026-04-16 14:33:37.800058684 +0000
+++ ./output/git_project/diff_cache/8/Lab4.java	2026-04-16 14:36:40.000000000 +0000
@@ -19,8 +19,5 @@
+previous : 6
+previous : 7
\end{lstlisting}

В процессе создания ревизии в \texttt{Subversion} возникли аналогичные конфликты:

\begin{lstlisting}
#viewing conflicts of output/svn_project/local/Blue/Lab4.java
Line 22: <<<<<<< .working
Line 23: previous : 3
Line 24: ||||||| .merge-left.r5
Line 25: previous : 0
Line 26: =======
Line 27: previous : 1
Line 28: >>>>>>> .merge-right.r9
\end{lstlisting}

Поступим схожим образом, сверившись с предоставленным снимком репозитория. Разрешим их следующим образом:

\begin{lstlisting}
--- ./output/svn_project/local/Blue/Lab4.java	2026-04-16 14:33:43.944058736 +0000
+++ ./output/svn_project/diff_cache/8/Lab4.java	2026-04-16 14:36:46.000000000 +0000
@@ -19,10 +19,5 @@
+previous : 6
+previous : 7
\end{lstlisting}
