Согласно п. 3.3.2 кодекса обучающегося Университета ИТМО, использование материалов данного репозитория _(кода, логики генерации или текста отчёта)_ в ваших работах будет расценено как **плагиат**. Это влечёт за собой:

## Команды Makefile

В проекте используется `Makefile` для автоматизации рутинных задач. Основные цели:

* **clean**: удаляет временные файлы компиляции и артефакты проекта
* **generate**: запускает логику формирования артефактов VCS через `src/main.py`
* **pdf**: выполняет двойную компиляцию исходного кода через `xelatex` для корректного отображения ссылок и оглавления

## Процесс сборки

Сборка документации выполняется в два этапа: генерация контента и компиляция PDF.

1. Используйте образ `2-vcs-environment` для очистки старых артефактов и запуска Python-скриптов генерации:
```bash
docker run --rm -v .:/project ghcr.io/itmo-fse-ld-2026/2-vcs-environment:latest make clean generate
```

2. Используйте образ `2-xetex-compiler` для финальной компиляции документа:
```bash
docker run --rm -v .:/data ghcr.io/itmo-fse-ld-2026/2-xetex-compiler:latest make pdf
```

## Конфигурационный файл

Файл `config.yaml` управляет параметрами генерации артефактов и отчёта.

| Ключ | Тип | Описание |
|---|---|---|
| `variant` | `int` | Номер варианта лабораторной работы. |
| `ask_commit_messages` | `bool` | Запрашивать ли сообщения коммитов интерактивно. Если `false`, используются значения по умолчанию. |
| `base_url` | `string` | Базовый URL для доступа к API генерации вариантов текущей лабораторной работы. |
| `report_dir` | `string` | Путь к директории с материалами отчёта. |
| `output_dir` | `string` | Путь к директории, куда складываются сгенерированные артефакты. |
| `git_dir` | `string` | Относительный путь директории локального Git-проекта. |
| `svn_dir` | `string` | Относительный путь директории локального Subversion-проекта. |
| `git_log` | `string` | Имя файла с логом команд Git. |
| `svn_log` | `string` | Имя файла с логом команд Subversion. |
| `git_err` | `string` | Имя файла с обнаруженными конфликтами слияния в Git. |
| `svn_err` | `string` | Имя файла с обнаруженными конфликтами слияния в Subversion. |
| `vcs_plot` | `string` | Относительный путь к выходному `.tex`-файлу _(TikZ)_ с блок-схемой выданного варианта. |
