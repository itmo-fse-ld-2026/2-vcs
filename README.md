## Команды Makefile

В проекте используется `Makefile` для автоматизации рутинных задач. Основные цели:

* **clean**: удаляет временные файлы компиляции и артефакты проекта
* **generate**: запускает логику формирования артефактов VCS через `src/main.py`
* **pdf**: выполняет двойную компиляцию исходного кода через `xelatex` для корректного отображения ссылок и оглавления

## Процесс сборки

Сборка документации выполняется в два этапа: генерация контента и компиляция PDF.

1. Используйте образ `2-vcs-environment` для очистки старых артефактов и запуска Python-скриптов генерации:
```bash
docker run --rm -v .:/project ghcr.io/itmo-fse-ld-2026/2-vcs-environment:latest make clean generate
```

2. Используйте образ `2-xetex-compiler` для финальной компиляции документа:
```bash
docker run --rm -v .:/data ghcr.io/itmo-fse-ld-2026/2-xetex-compiler:latest make pdf
```
